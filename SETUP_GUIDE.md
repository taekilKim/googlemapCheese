# 🔑 Google Maps API 키 발급 가이드 (초보자용)

## 단계 1: Google Cloud Console 접속

1. **브라우저에서 Google Cloud Console 열기**
   ```
   https://console.cloud.google.com
   ```

2. **Google 계정으로 로그인**
   - Gmail 계정 사용 (없으면 생성)
   - 개인 계정이든 업무용 계정이든 상관없음

---

## 단계 2: 프로젝트 생성

1. **상단 프로젝트 선택 드롭다운 클릭**
   - 화면 왼쪽 상단에 "Google Cloud" 로고 옆에 프로젝트 이름이 보임
   - 처음이면 "프로젝트 선택" 또는 "Select a project"라고 표시됨

2. **"새 프로젝트" 버튼 클릭**
   - 팝업 창 오른쪽 상단의 "새 프로젝트(NEW PROJECT)" 클릭

3. **프로젝트 정보 입력**
   - **프로젝트 이름**: `GoogleMapsPhotoMaker` (또는 원하는 이름)
   - **위치**: 조직 없음 (개인 사용이면 그대로 두기)
   - **"만들기"** 버튼 클릭

4. **프로젝트 생성 대기**
   - 30초~1분 정도 소요
   - 화면 우측 상단 알림 벨 아이콘에서 진행 상태 확인

---

## 단계 3: Places API 활성화

1. **좌측 메뉴 열기**
   - 왼쪽 상단 햄버거 메뉴(≡) 클릭

2. **API 및 서비스 > 라이브러리 선택**
   ```
   ≡ 메뉴 → API 및 서비스 → 라이브러리
   ```

3. **Places API 검색**
   - 검색창에 `Places API` 입력
   - "Places API" 클릭 (Google Maps Platform 제품)

4. **API 활성화**
   - **"사용 설정"** 또는 **"ENABLE"** 버튼 클릭
   - 몇 초 후 "API 사용 설정됨" 메시지 표시

---

## 단계 4: 결제 계정 설정 (무료 크레딧 사용)

**⚠️ 중요: 결제 정보를 입력해야 하지만, 무료 범위 내에서는 요금이 청구되지 않습니다!**

1. **"API 사용 설정" 후 결제 계정 요청 팝업이 나타날 수 있음**
   - "결제 계정 설정" 또는 "Set up billing account" 클릭

2. **결제 정보 입력**
   - 국가: 대한민국
   - 카드 정보 입력 (신용/체크카드)
   - 주소 입력

3. **무료 크레딧 확인**
   - ✅ 매월 **$200 무료 크레딧** 제공
   - ✅ Places API Details: $0.017/건
   - ✅ 즉, **매월 약 11,700회 무료**
   - ✅ 일반 개인 사용에는 절대 유료 전환 안 됨

4. **예산 알림 설정 (선택사항, 권장)**
   - 좌측 메뉴 → 결제 → 예산 및 알림
   - "예산 만들기" 클릭
   - 금액: $50 설정
   - 알림: 50%, 90%, 100% 도달 시 이메일
   - 이렇게 하면 만약 무료 범위를 초과하면 이메일로 알림 옴

---

## 단계 5: API 키 생성

1. **좌측 메뉴에서 "사용자 인증 정보" 선택**
   ```
   ≡ 메뉴 → API 및 서비스 → 사용자 인증 정보
   ```

2. **"+ 사용자 인증 정보 만들기" 클릭**
   - 화면 상단의 "+ 사용자 인증 정보 만들기" 버튼
   - 드롭다운에서 **"API 키"** 선택

3. **API 키 생성 완료**
   - 팝업에 API 키가 표시됨
   - 예시: `AIzaSyDXXXXXXXXXXXXXXXXXXXXXXXXXXXX`
   - **"키 복사"** 버튼 클릭하여 복사
   - ⚠️ 안전한 곳에 보관 (나중에도 확인 가능하지만)

---

## 단계 6: API 키 보안 설정 (필수!)

**⚠️ 중요: API 키를 보호하지 않으면 다른 사람이 악용할 수 있습니다!**

1. **방금 생성된 API 키 옆의 연필 아이콘(편집) 클릭**

2. **"애플리케이션 제한사항" 설정**
   - **개발/테스트 중**: "없음" 선택 (나중에 변경)
   - **배포 후**: "HTTP 리퍼러(웹사이트)" 선택
     - 웹사이트 제한사항 추가:
       ```
       https://yourdomain.com/*
       https://*.vercel.app/*  (Vercel 사용 시)
       localhost:3000/*  (로컬 테스트용)
       ```

3. **"API 제한사항" 설정**
   - **"키 제한"** 선택
   - **"Places API"** 체크박스만 선택
   - 다른 API는 모두 해제 (보안 강화)

4. **일일 할당량 제한 (선택사항, 권장)**
   - 좌측 메뉴 → API 및 서비스 → 사용자 인증 정보
   - API 키 클릭 → "할당량" 탭
   - 일일 요청 수 제한: 1000건 (또는 원하는 값)
   - 악의적인 사용 방지

5. **"저장" 버튼 클릭**

---

## 단계 7: API 키 확인 및 테스트

1. **API 키 목록에서 확인**
   - 사용자 인증 정보 페이지에서 생성한 API 키 확인
   - 이름을 클릭하면 언제든지 키 확인 가능

2. **API 키 복사**
   - 키를 복사해서 안전한 곳에 보관
   - 예시: `AIzaSyABCDEFGHIJKLMNOPQRSTUVWXYZ1234567`

## 💡 팁

### API 사용량 모니터링
```
Google Cloud Console → API 및 서비스 → 대시보드
```
- 일일/주간/월간 사용량 확인
- 차트로 시각화
- 무료 크레딧 잔액 확인

### API 키 재생성
혹시 키가 유출되었다면:
1. 사용자 인증 정보 → API 키 선택
2. "키 재생성" 클릭
3. 이전 키는 즉시 무효화됨

### 여러 환경용 키 분리
- **개발용 키**: 제한사항 없음
- **프로덕션용 키**: HTTP 리퍼러 제한
- 각각 별도로 생성 권장

## 🆘 문제 해결

### "이 API에는 결제 계정이 필요합니다" 오류
→ 결제 정보를 입력해야 함 (무료 크레딧 사용 위해)

### "API가 활성화되지 않음" 오류
→ Places API가 활성화되었는지 확인

### "할당량 초과" 오류
→ Google Cloud Console에서 사용량 확인

### API 키가 작동하지 않음
→ 키 생성 후 몇 분 기다려보기 (전파 시간 필요)
